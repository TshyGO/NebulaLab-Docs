# Roadmap

Nebula Lab 的长期方向是成为实验工作流平台，而不是完整复制机构级科研管理系统。

## 当前主线

当前版本聚焦两个基础方向：

- 实验流程处理：数据导入、处理、绘图、导出和流程复用。
- 社区构建：插件、模板、主题和资源分享。

这两个方向会继续维护和升级，并作为后续实验记录平台的基础。

## 下一阶段

后续计划围绕完整实验链路展开：

```text
配方计算 -> 配方保存 -> 实验记录 -> 原始数据导入 -> 数据处理 -> 结果写入 -> 历史检索
```

这条链路会让一次实验从配方和样品开始，逐步沉淀为可追溯、可复用的数据资产。

## 长期目标

长期目标是支持课题组自部署的实验数据平台：

- 桌面端继续负责本地数据处理、绘图和插件运行。
- 课题组服务器负责账号、权限、实验记录、文件索引、共享和备份。
- 数据库保留配方、工艺、样品、原始数据、处理流程、结果和结论之间的关系。

当课题组积累足够多的一手实验数据后，Nebula Lab 可以进一步支持材料数据检索、数据合集、AI 问答和材料基因库基础能力。
