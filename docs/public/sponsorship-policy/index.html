    统计口径

    本机制刻意避免人工打分。它只使用 GitHub 可追溯数据，并通过分级权重 + 周聚合 + sqrt 折算降低机械改动对分配结果的影响。

    只算已合并贡献

    统计主分支中的非 merge commit。未合并分支、实验代码和废弃改动不进入结算。

    排除低信号文件

    lock / min / map / dist / vendor 等默认权重 0；未分类路径 fallback 也是 0，避免依赖更新或大型数据导入放大比例。

    按周开方

    把同一作者一周内的 weighted churn 累加后再 sqrt，commit 切片刷分自动失效。

    Co-authored-by 均分

    Squash merge 时 GitHub 自动写入 Co-authored-by trailer；脚本会把 weighted churn 在所有贡献者间均分。

    非秒级实时

    页面筛选即时重算；底层数据由 GitHub Actions 在 main 更新、每日定时或手动运行时重新生成并部署。

    跨仓库权重

    主仓库 1.0、插件库 0.9、Docs 0.45。仓库筛选只影响查看口径，默认结算看全部仓库。

    file_weighted_churn = (additions + 0.5 × deletions) × file_type_weight
bucket_churn(author, week, repo) = Σ (1 / N_authors) × file_weighted_churn
bucket_score(author, week, repo) = sqrt(bucket_churn) × repo_weight
个人积分 = Σ over (week, repo) of bucket_score(self, week, repo)
个人分配比例 = 个人积分 / 全部贡献者积分之和
